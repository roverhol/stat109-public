Step 2 — Make the three “canonical maps” (2–3 hours)

Create concepts/learning-objectives.yml (IDs + text).

Create methods.qmd (convert your 2-page table; add anchors).

Create r-reference.qmd (paste base-R cheat sheet; add anchors).

Step 3 — Build your lesson template (30 min)

Create one file lessons/L00-template.qmd (not linked on site).

Include the flow sections: Physical → Simulation → Math+Conditions → Hand → R → Practice.

Step 4 — Convert Weeks 1–3 (most important) (one focused day)

For each of the first 3 lecture outlines:

Convert OneNote outline → lessons/L01.qmd, L02.qmd, L03.qmd.

Add YAML: lesson_id, learning_objectives, formats (html+pdf).

Keep content concise; preserve whitespace for work.

Add a “Practice” list with problem IDs (even if those problems aren’t converted yet).

Step 5 — Seed the problem bank + quiz 1 (half day)

Convert 10–15 practice problems into problems/ with IDs + LO tags.

Convert solutions for the 5–8 you’ll reuse most soon.

Create quizzes/QZ-01.qmd listing problem IDs.

Step 6 — Convert Lab 1 into a notebook (2–4 hours)

Take your R script → labs/LAB-01.qmd.

Add narrative prompts between chunks.

Link it from Lesson 1 and/or Schedule.

Step 7 — Projects: export + publish Project 1 (2–3 hours)

Export the Canvas rubric for Project 1 into projects/raw/.

Create projects/PRJ-01.qmd and projects/PRJ-01-rubric.qmd.

Put only the submission link in Canvas; keep everything else on the site.

Step 8 — Build the schedule hub (1–2 hours)

In schedule.qmd, create a table for Weeks 1–15.

Each week links to: lesson(s), lab, quiz, project (when applicable), readings.
