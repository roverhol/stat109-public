Lab 13: Project 5 work day (Google Sheets -> R -> prediction)

NoteOpen in Google Colab (R)

Open in Colab — after it opens, use Runtime -> Change runtime type and select R if needed.

Learning outcomes

By the end of this lab, you should be able to:

  1. Connect a Google Sheet to R using a CSV export link.
  2. Load data in Colab with readr::read_csv() (same pattern as TidyTuesday URLs).
  3. Fit a simple linear regression model with lm(response ~ predictor).
  4. Print and interpret model fit output (summary(), coef()).
  5. Make a point prediction and a confidence interval for the mean response using predict().
  6. Create and interpret a scatterplot with fitted line.

Quick RStudio refresher (very short)

If you are running this outside Colab in RStudio:

  • Script/Source pane (top-left): where you write and save code (.R script or .qmd).
  • Console (bottom-left): where code runs and output appears.
  • Terminal tab (bottom-left, next to Console): shell commands (ls, git status, etc.) if needed.
  • Environment/Files/Plots pane (right): inspect objects, files, and figures.

Very short Quarto (.qmd) demo

A practical next step from .ipynb is a .qmd file for a more reproducible workflow (same code + clean rendered output such as PDF/HTML).

Motivation: ultimately, in statistics we usually need to deliver a clear document/PDF/slides that communicates results. Quarto is a strong workflow for this because it keeps a direct connection between your final output, your analysis code, and your dataset. That reproducible link makes your work easier to check, update, and trust.

# Mini analysis

```r
# Quarto chunk options would normally go at the top of a real chunk, e.g.:
# #| echo: false
# #| fig-cap: "Scatterplot of response vs predictor."
# #| fig-width: 5
# #| fig-height: 3.5
library(tidyverse)
dat <- readr::read_csv("https://docs.google.com/spreadsheets/d/YOUR_SHEET_ID/export?format=csv&gid=0")
ggplot(dat, aes(predictor, response)) + geom_point()
```

What these pieces mean

  • YAML (the top --- block): document metadata and output format.
  • Chunk option lines (#| ...): instructions for rendering (caption, plot size, hide/show code, etc.).
  • echo: false hides code but still shows output/figure.
  • Use echo: true when you want students/readers to see code.

Render to PDF

From Terminal:

quarto render your-file.qmd --to pdf

## 4) Common troubleshooting

- **403/permission error:** sheet is not shared for viewing.
- **Wrong sheet tab:** `gid` in URL points to a different tab.
- **Column name mismatch:** check exact header spelling with `names(dat)`.
- **Non-numeric columns:** convert with `as.numeric()` only after checking raw values/units.
