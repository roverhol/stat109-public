Lab 9: Introduction to the tidyverse

This lab introduces tidyverse workflows for describing one variable at a time, then using filter() and group_by() for focused summaries.

Learning outcomes

By the end of this lab, you should be able to:

  1. Load a tidyverse dataset and inspect its variables.
  2. Identify whether a variable is numerical or categorical.
  3. Check category levels for categorical variables.
  4. Recode a variable type when numeric values are actually category codes.
  5. For one numerical variable, compute mean and SD and make a histogram.
  6. For one categorical variable, compute count and proportion.
  7. Use filter() to subset data with conditions.
  8. Use group_by() with summarise() to compare groups.

Summary of new functions

Function What it does
library(tidyverse) Loads the core tidyverse packages
glimpse(df) Quick view of variable names and types
class(x) Reports object type
levels(x) Shows categories if x is a factor
mutate() Adds/changes variables
factor() Converts a variable to categorical type
summarise() Computes summary statistics
n() Counts rows inside summarise()
filter() Keeps rows that satisfy conditions
group_by() Creates groups before summaries
ggplot() + geom_histogram() Makes a histogram for numerical data
count() Quick count by category

Setup

Code 
if (!requireNamespace("tidyverse", quietly = TRUE)) {
  install.packages("tidyverse", repos = "https://cloud.r-project.org")
}

library(tidyverse)

We’ll use the built-in mpg dataset from ggplot2 (loaded with tidyverse). Each row is a car model with variables like city MPG (cty), highway MPG (hwy), drive type (drv), and class (class).

Code 
data("mpg", package = "ggplot2")
glimpse(mpg)

Part 1: One numerical variable

Let’s describe hwy (highway MPG), a numerical variable.

1A) Check type

Code 
class(mpg$hwy)

1B) Compute mean and SD

Code 
mpg %>%
  summarise(
    mean_hwy = mean(hwy),
    sd_hwy = sd(hwy)
  )

1C) Make a histogram

Code 
ggplot(mpg, aes(x = hwy)) +
  geom_histogram(binwidth = 2, boundary = 0, color = "white") +
  labs(
    title = "Distribution of highway MPG",
    x = "Highway MPG (hwy)",
    y = "Count"
  )

Part 2: One categorical variable

Now describe drv (drive type), a categorical variable.

2A) Check type and categories

Code 
class(mpg$drv)
levels(mpg$drv)

2B) Compute counts and proportions

Code 
mpg %>%
  count(drv, name = "count") %>%
  mutate(proportion = count / sum(count))

Part 3: Change variable type (numeric codes -> categories)

In mpg, cyl (number of cylinders) is numeric, but in many analyses we treat it as categories (4, 5, 6, 8) rather than a continuous scale.

Code 
class(mpg$cyl)

Convert cyl to a factor:

Code 
mpg2 <- mpg %>%
  mutate(cyl = factor(cyl))

class(mpg2$cyl)
levels(mpg2$cyl)

Now compute count and proportion for cyl categories:

Code 
mpg2 %>%
  count(cyl, name = "count") %>%
  mutate(proportion = count / sum(count))

Part 4: filter() with conditions

Use filter() to keep only rows meeting conditions.

Examples:

Code 
# Cars with highway MPG at least 30
mpg %>%
  filter(hwy >= 30) %>%
  select(manufacturer, model, hwy) %>%
  head()
Code 
# SUVs only
mpg %>%
  filter(class == "suv") %>%
  select(manufacturer, model, class, hwy) %>%
  head()
Code 
# SUVs with 4-wheel drive
mpg %>%
  filter(class == "suv", drv == "4") %>%
  select(manufacturer, model, class, drv, hwy) %>%
  head()

Part 5: group_by() + summarise()

Now summarize a numerical variable by groups.

5A) Mean and SD of highway MPG by drive type

Code 
mpg %>%
  group_by(drv) %>%
  summarise(
    n = n(),
    mean_hwy = mean(hwy),
    sd_hwy = sd(hwy)
  )

5B) Mean and SD of highway MPG by vehicle class

Code 
mpg %>%
  group_by(class) %>%
  summarise(
    n = n(),
    mean_hwy = mean(hwy),
    sd_hwy = sd(hwy)
  ) %>%
  arrange(desc(mean_hwy))

Practice tasks (student work)

Use mpg to complete the following:

  1. Pick one numerical variable (cty, hwy, or displ) and report mean + SD.
  2. Make a histogram for that variable.
  3. Pick one categorical variable (drv, class, or manufacturer) and report count + proportion.
  4. Use filter() to keep rows with hwy >= 28 and drv == "f". Show first 10 rows of manufacturer, model, hwy, and drv.
  5. Use group_by(class) and report mean cty by class.
  6. Convert one coded variable to factor (for example cyl) and list its levels.

Wrap-up

Today you practiced a tidyverse workflow for one-variable description:

  • identify variable type,
  • compute the right summary for that type,
  • visualize numerical variables,
  • and apply filter() and group_by() for focused comparisons.

These steps will be reused throughout project work and upcoming labs.
