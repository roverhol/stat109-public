Two-sample tests and confidence intervals

Use the Lecture 30 Methods table to pick one primary test for the “by chance” question.

Response type R command Notes
Numerical, 2 groups t.test(y ~ group, data = dat, var.equal = FALSE) Welch t-test; output includes 95% CI for \(\mu_1 - \mu_2\) (Lecture 29)
Binary (0/1 or two levels), 2 groups prop.test(x = c(x1, x2), n = c(n1, n2), correct = FALSE) Compares two proportions
Two categorical variables chisq.test(tab) Check chisq.test(tab)$expected for ≥ 5 rule (Lecture 28)

# R Quick Reference Guide

This page summarizes R code from the tidyverse from our class.

------------------------------------------------------------------------

## Analysis Order {#r-pipeline}

| Step | Code idea | Quick check |
|------------------|-------------------------|-----------------------------|
| 1 | `library(tidyverse)` | Packages load without errors |
| 2 | `read_csv(...); glimpse(dat)` | Column **names** and **types** match **Part 1** |
| 3 | `mutate(... factor(..., levels = ...))` | **Levels** match your two groups / outcome categories |
| 4 | `group_by` + `summarise` and/or `count` | **Means/SDs** if response is numerical; **counts** (and optional props) if categorical |
| 5 | **`ggplot`** | **Boxplot/histogram** for numerical response by group; **bar chart** of counts for binary/categorical (**L25–27**) |
| 6 | **Test + CI** | **`t.test(..., var.equal = FALSE)`** *or* **`prop.test` / `chisq.test`** per Methods table |
| 7 | Short **interpretation** in prose | *p*-value addresses **chance** only; causal language only if design supports it |

------------------------------------------------------------------------

## Summaries {#r-summaries}

| Setting | Code pattern |
|---------------------------|---------------------------------------------|
| One numerical variable | `dat \|> summarize(mean_y = mean(y), sd_y = sd(y), n_y=n())` |
| Numerical **by two groups** | `dat \|> group_by(group) \|> summarize(n = n(), mean_y = mean(y), sd_y = sd(y))` |
| Counts for one categorical | `dat \|> count(category) \|> mutate(prop = n / sum(n))` |
| **2 × 2** counts | `dat \|> count(group, outcome)` then `pivot_wider` |

------------------------------------------------------------------------

## Plots (ggplot2) {#r-plots}

| Setting | Pattern |
|-------------------------|-----------------------------------------------|
| **Numerical** response, **two groups** | `ggplot(dat, aes(x = group, y = y)) + geom_boxplot()` (**Lecture 25**); or `geom_histogram(aes(fill = group), ...)` |
| **Two categorical** (e.g. group × outcome) | `dat \|> count(group, outcome) \|> ggplot(aes(x = group, y = n, fill = outcome)) + geom_col()` (**Lecture 27**); add `position = "fill"` for 100% stacked |
| **Two numerical** | `ggplot(dat, aes(x = x1, y = x2)) + geom_point()` (**Lecture 26**) |
| **Subclassification** (confounder) | Add `facet_wrap(~ confounder)` to the plot above (**Lecture 28**) |

Always add **`labs(title = ..., x = ..., y = ...)`** with **units** where relevant.

------------------------------------------------------------------------

## Two-sample tests and confidence intervals {#r-tests}

Use the **Lecture 30 Methods table** to pick **one** primary test for the **“by chance”** question.

| Response type | R command | Notes |
|-------------------------------|-----------------------|------------------|
| **Numerical**, 2 groups | `t.test(y ~ group, data = dat, var.equal = FALSE)` | **Welch** *t*-test; output includes **95% CI** for $\mu_1 - \mu_2$ (**Lecture 29**) |
| **Binary** (0/1 or two levels), 2 groups | `prop.test(x = c(x1, x2), n = c(n1, n2), correct = FALSE)` | Compares two **proportions** |
| Two categorical variables | `chisq.test(tab)` | Check **`chisq.test(tab)$expected`** for **≥ 5** rule (**Lecture 28**) |

------------------------------------------------------------------------

## Simulating small datasets {#r-sim}

| Task | Example pattern |

| Binary outcome | `rbinom(n, 1, prob = ...)` inside `mutate` or `c(...)` |
| Save | `write_csv(dat, "project4_data.csv")` |

1.  **`read_csv`** path and **`glimpse`** — Do column names match your **Part 1** definitions?

2.  **`factor`** — Are **group** and **outcome** coded with the **correct levels**?

3.  **`group_by` + `summarize`** — Does the code compute **means/SDs** for **numerical** responses and **counts/proportions** for **categorical** responses?

4.  **`ggplot`** — Is the **geom** right (**boxplot** vs **bar** vs **histogram**) for your variable types?

5.  **Test** — Is it **`t.test(..., var.equal = FALSE)`** for mean vs **`prop.test` / `chisq.test`** for proportions?

6.  **Conditions** — random sample or assignment? sample size large enough? See methods table for exact condition to check.
