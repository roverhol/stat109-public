STAT 109: Introductory Biostatistics - Simple logistic regression

Lecture 37

From normal to binomial: a simple logistic regression model

For a binary outcome (0/1, No/Yes), we replace normal-errors linear regression with a binomial model:

\[\text{logit}(p) = \beta_0 + \beta_1 x,\ \ \ p = P(Y=1|X=x)\]

Fit in R with:

  • glm(y ~ x, family = binomial, data = ...)

Main prediction target is probability p, then classify using a cutoff (often 0.5 unless context suggests otherwise).

Learning objectives

By the end of this lecture, you should be able to:

  • explain why logistic regression is used for binary outcomes;
  • fit and interpret a simple logistic regression model with glm(..., family = binomial);
  • interpret the slope as a change in log-odds (and direction of odds change);
  • compute and interpret a confidence interval for the logistic slope;
  • generate predicted probabilities and classify outcomes using a cutoff;
  • compare training and test accuracy and interpret out-of-sample classification performance.

Example 1 - Study hours and pass/fail

Code 
if (!requireNamespace("tidyverse", quietly = TRUE)) {
  install.packages("tidyverse", repos = "https://cloud.r-project.org")
}

library(tidyverse)

set.seed(3601)
n <- 160
pass_dat <- tibble(
  study_hours = runif(n, 0, 15)
) |>
  mutate(
    p_pass = plogis(-2.4 + 0.38 * study_hours),
    pass = rbinom(n(), 1, p_pass)
  )

Train/test split and fit

Code 
set.seed(3602)
idx <- sample(seq_len(nrow(pass_dat)), size = 0.8 * nrow(pass_dat))
pass_train <- pass_dat[idx, ]
pass_test  <- pass_dat[-idx, ]

fit_pass <- glm(pass ~ study_hours, family = binomial, data = pass_train)
summary(fit_pass)

Predicted probability and cutoff-based class prediction

Code 
pass_train <- pass_train |>
  mutate(
    p_hat = predict(fit_pass, newdata = pass_train, type = "response"),
    y_hat_50 = if_else(p_hat >= 0.5, 1, 0)
  )

pass_test <- pass_test |>
  mutate(
    p_hat = predict(fit_pass, newdata = pass_test, type = "response"),
    y_hat_50 = if_else(p_hat >= 0.5, 1, 0)
  )

Accuracy on training and test

Code 
acc <- function(y, yhat) mean(y == yhat)

tibble(
  set = c("train", "test"),
  accuracy = c(
    acc(pass_train$pass, pass_train$y_hat_50),
    acc(pass_test$pass, pass_test$y_hat_50)
  )
)
# A tibble: 2 × 2

Training accuracy can be optimistic; test accuracy better represents performance on new data.

Key takeaways

  • Logistic regression models probability for a binary outcome via the logit link.
  • Use training data to fit the model; report test accuracy for generalization.
  • Slope CI tells whether the log-odds effect is plausibly positive, negative, or near zero.
  • Probability prediction comes first; class prediction depends on chosen cutoff.
